# 生成AIはどこを改善すると賢くなるのか

生成AIの性能向上は、単にCPUやメモリを増やすことではない。主に次の要素を組み合わせて実現される。

| 改善する場所 | 代表的な改善 | 主に良くなること |
| --- | --- | --- |
| 学習データ | 量、品質、多様性、重複除去、誤情報の削減 | 知識、言語能力、正確さ |
| モデル | パラメータ数、構造、長い文脈への対応 | 表現力、推論力、扱える情報量 |
| 学習方法 | 学習量の配分、指示学習、人間・AIのフィードバック | 指示への従いやすさ、安全性、回答の有用性 |
| 回答時の処理 | 検索、ツール利用、複数案の比較、推論時間の追加 | 最新情報、計算の正確さ、難問への対応 |
| ハードウェア | GPU、VRAM、RAM、通信速度 | より大きな学習や長い入力を実行できること、速度、コスト |

つまり、ハードウェアは「より良い学習や回答方法を実行するための土台」であり、容量を増やしただけで、すでに完成している同じモデルの知識や判断力が自動的に増えるわけではない。

## まず押さえる全体像

現在の文章生成AIの多くは、大規模言語モデル（LLM）を中心に構成されている。LLMは、入力された文章に続く次の単語の断片を確率的に予測し、それを繰り返して回答を作る。

```text
大量の学習データ
      ↓
事前学習: 続きの予測を何度も練習する
      ↓
学習済みモデル: パラメータに言葉の関係やパターンが記録される
      ↓
追加学習: 指示への従い方や望ましい回答を学ぶ
      ↓
ユーザーの入力 → 必要なら検索・ツール・推論 → 回答
```

「学習」と「回答」は別の段階である。

- 学習時は、大量の文章を使ってモデル内部のパラメータを調整する。
- 回答時は、原則としてパラメータを変更せず、学習済みの能力と、その場で渡された情報を使う。

## 回答が作られる簡単な仕組み

### 1. 文章をトークンに分ける

入力文を、単語や文字そのものではなく、処理しやすい小さな単位である「トークン」に分ける。日本語では、1文字、単語の一部、記号などが一つのトークンになる場合がある。

### 2. トークンを数値に変換する

各トークンをベクトルと呼ばれる数値の並びに変換する。これにより、コンピューターが言葉同士の関係を計算できるようになる。

### 3. Transformerで文脈を計算する

現在のLLMの中心には、Transformerという構造がある。TransformerのAttentionは、入力中のどの部分をどの程度参照するかを計算する。たとえば代名詞が何を指すか、質問のどの条件が重要か、といった関係を扱う土台になる。

Transformerは計算を並列化しやすく、大規模な学習に向いていたことも、現在の生成AIの発展につながった。

### 4. 次のトークンの確率を出す

モデルは「次にどのトークンが来そうか」という確率分布を出す。その中から一つを選び、入力の末尾に追加して、また次を予測する。この処理を回答の終わりまで繰り返す。

モデルは文章をデータベースからそのまま取り出しているわけではない。学習で得たパターンから、その文脈に合いそうな続きを生成する。そのため、自然な文章でも事実が誤っている「ハルシネーション」が起こり得る。

## 学習では何をしているのか

### 事前学習

大量の文章やコードなどを読み、伏せられた部分や次のトークンを予測する。予測と正解のずれを数値化し、そのずれが小さくなるようにパラメータを少しずつ更新する。

パラメータはモデル内部にある多数の数値であり、学習によって調整される。特定の文をそのまま保存する「記憶容量」とは性質が異なり、言葉、概念、文脈の関係を生成に利用できる形で圧縮して持つ。

### 追加学習

事前学習だけでは、「文章の続きを作る能力」はあっても、ユーザーの依頼に従い、安全で役立つ回答を返すとは限らない。そこで、次のような追加学習を行う。

- 指示と望ましい回答の組み合わせを使う指示学習
- 複数の回答に人間や別のモデルが評価を付け、その好みを学ぶ方法
- 数学、コード、特定業務などに適応させる専門データでの学習

人間のフィードバックを使った研究では、単にモデルを大きくするだけでなく、望ましい振る舞いを学ばせることが回答品質に重要だと示されている。

## 性能が上がる主なポイント

### 1. 学習データを改善する

データは、量だけでなく中身が重要である。

| 改善例 | 期待できる効果 |
| --- | --- |
| データ量を増やす | より多くの表現、知識、問題パターンを学べる |
| 誤情報や低品質文を減らす | 不正確、不自然、無意味な出力を減らしやすい |
| 重複を除く | 同じ内容への過度な偏りや丸暗記を抑えやすい |
| 分野や言語の偏りを直す | 対応できる利用者、言語、専門分野が広がる |
| 難易度や順序を調整する | 学習を安定させ、必要な能力を身につけやすくする |
| 指示と良い回答のデータを整える | ユーザーの意図に沿った形式で答えやすくなる |

ただし、データを増やせば無条件で良くなるわけではない。モデルの大きさに対して学習データが少なすぎれば能力を十分に引き出せず、低品質なデータを大量に加えると誤りや偏りも学習する。

Chinchillaの研究では、一定の計算量の中でモデルのパラメータ数だけを増やすより、モデルサイズと学習トークン数の配分を適切にすることが重要だと報告された。小さいモデルでも、十分なデータで学習したことで、より大きい比較対象を上回った。

### 2. モデルの大きさを改善する

一般にパラメータ数を増やすと、より複雑なパターンを表現できる。ただし、次の条件も必要になる。

- 規模に見合う量と質の学習データ
- 学習を安定させる方法
- 学習と回答を実行できる計算資源
- 実際の利用目的に合った評価と追加学習

研究では、モデルサイズ、データ量、学習に使う計算量を増やすと、一定範囲で予測誤差が規則的に小さくなる傾向が確認されている。一方、「パラメータが多いほど、常にユーザーにとって良い回答になる」という意味ではない。

### 3. モデルの構造を改善する

同じような計算量でも、情報の扱い方を変えることで性能や効率を上げられる。

- Attentionの計算方法を改良する
- 長い文章を効率良く扱えるようにする
- 入力に応じて一部の専門ネットワークだけを使う
- 文章だけでなく画像、音声、動画を同じモデルで扱う
- 学習時や回答時の数値精度を調整し、速度と品質を両立する

構造改善には、純粋な正答率を上げるものだけでなく、同じ品質をより少ない計算量やメモリで実現するものもある。安く速く提供できれば、より長い入力や多くの試行に計算資源を回せるため、システム全体の出力品質にもつながる。

### 4. 学習方法を改善する

同じモデルとデータでも、学習のさせ方によって結果は変わる。

- どの程度の速さでパラメータを更新するか
- どのデータをどの割合、順番で見せるか
- 正解だけでなく、問題を解く途中の過程も学ばせるか
- 人間や別のAIによる評価をどう反映するか
- 誤答、危険な回答、指示違反をどのように減らすか

ここが改善されると、知識量が同程度でも、質問の意図を読み、指定形式を守り、分かりやすく答える能力が上がる。

### 5. 回答時に使う計算を増やす

学習済みモデルを変更しなくても、回答時の処理を工夫すると結果を改善できる。

- 難しい問題を複数の段階に分ける
- 複数の回答候補を作り、検証して選ぶ
- 計算機、コード実行、データベースなどのツールを使う
- 検索結果や社内文書を取得し、根拠として入力に加える
- 問題の難しさに応じて、回答に使う計算時間を増減する

推論過程を段階化する方法は、算術や記号的推論などで性能を改善できることが報告されている。また、回答時の計算量を問題ごとに適切に配分する研究では、一定条件下で、単にモデルを大型化するより効率良く正答率を上げられる場合が示されている。

### 6. 外部情報とツールを改善する

モデル内部の知識には、学習時点、正確さ、情報量の限界がある。そこで、回答前に関連文書を検索して入力へ追加するRAG（Retrieval-Augmented Generation）や、外部ツールを使う仕組みが利用される。

これにより、モデルを再学習しなくても次の改善が可能になる。

- 最新情報や社内固有情報を参照する
- 回答の根拠を示す
- 電卓やプログラムで厳密に計算する
- 実際のファイル、予定、データベースを確認する

ただし、検索結果が間違っている、必要な情報を取得できない、取得した情報の読み取りを誤る、といった別の失敗は残る。

## CPU、GPU、メモリを増やすと何が変わるか

### CPU

CPUは、データの読み込みや前処理、処理全体の制御、ツールの実行などを担う。CPUを高速化すると処理待ちが減るが、LLMの中心となる大規模な行列計算では、通常はGPUや専用アクセラレーターの影響が大きい。

### GPU・専用アクセラレーター

GPUは、多数の単純な計算を並列に実行するのが得意で、学習と回答の主要な計算を担う。

GPUを高速化したり台数を増やしたりすると、次が可能になる。

- より大きなモデルを学習する
- より多くのデータを学習する
- 実験回数を増やし、良い学習条件を探す
- 回答を速くする、または複数候補を検討する

これは性能向上を可能にするが、GPU自体が回答内容を理解して賢くするわけではない。

### VRAM

VRAMはGPUが直接使う高速なメモリで、主に次を保持する。

- モデルのパラメータ
- 計算途中の値
- 学習時にパラメータを更新するための情報
- 回答時に過去の入力を再利用するためのKVキャッシュ

VRAMを増やすと、より大きなモデル、長い入力、大きなバッチ、高い数値精度を扱いやすくなる。同じモデル、同じ入力、同じ計算方法をそのまま動かすだけなら、VRAM容量を増やしても回答の知識や推論能力は基本的に変わらない。

### RAM

RAMはCPU側の作業領域で、学習データ、前処理済みデータ、プログラム、モデルの一部などを保持する。容量を増やすと、大きなデータを扱いやすくなり、ストレージとの入れ替えを減らせる。

RAMもVRAMと同様に、容量そのものがモデルの能力になるわけではない。ただし、メモリ不足で実行できなかった学習方法や長い入力を扱えるようになれば、間接的に品質を上げられる。

### ストレージと通信

大量のデータやチェックポイントを高速に読み書きできるストレージと、多数のGPU間で計算結果を交換する高速な通信も重要である。ここが遅いと、高性能なGPUがデータ待ちになり、学習全体の効率が落ちる。

## 「メモリ」という言葉の違い

AIの話で使われる「メモリ」には複数の意味がある。

| 種類 | 意味 | 容量を増やしたときの主な効果 |
| --- | --- | --- |
| RAM・VRAM | コンピューターの一時的な作業領域 | 大きなモデルや長い入力を実行しやすくなる |
| パラメータ | 学習によって調整されたモデル内部の数値 | 表現できるパターンの容量が増える可能性がある |
| コンテキスト | 一回の回答でモデルが参照できる入力範囲 | 長い会話や文書を一度に扱いやすくなる |
| 外部メモリ | 検索対象の文書、データベース、過去情報 | 必要な情報を回答時に取り出せる |

これらは別物である。「会話を長く覚えられる」「学習済みの知識が多い」「PCのメモリが多い」は同じ意味ではない。

## 具体例で考える

### ケース1: VRAMだけを増やす

同じモデルを同じ設定で動かすなら、主に余裕と速度が改善し、回答内容はほぼ変わらない。より大きなモデルや長いコンテキストを使う設定へ変更して初めて、回答品質が変わる可能性がある。

### ケース2: 学習データを増やす

高品質で不足分野を補うデータなら性能向上が期待できる。一方、誤情報や重複をそのまま増やすと、改善しない場合や偏りが強くなる場合がある。

### ケース3: パラメータ数だけを増やす

表現力は増える可能性があるが、データや学習計算が不足すると能力を引き出せない。提供コストや回答時間も増えやすい。

### ケース4: 検索機能を追加する

モデル自体を変えなくても、最新情報や固有資料に基づく回答が可能になる。ただし、検索精度と、検索した内容を正しく使う能力の両方が必要になる。

### ケース5: 回答前に検証させる

候補を複数作る、計算をツールで確かめる、根拠と結論の矛盾を確認するといった処理により、難問の正答率を上げられる。ただし、時間と計算コストは増える。

## 性能は一つの数値ではない

「性能が高い」には複数の意味がある。

- 知識の広さと新しさ
- 事実の正確さ
- 論理、数学、コードの問題を解く力
- 指示や出力形式を守る力
- 長い文章を読み取る力
- 画像や音声を扱う力
- 安全性と偏りの少なさ
- 回答速度、同時利用者数、費用

ある改善がすべてを同時に良くするとは限らない。たとえば、長く考えさせれば難問には強くなる可能性があるが、回答は遅く高価になる。モデルを圧縮すれば速く安くなる一方、難しい問題の精度が落ちる場合がある。

そのため開発では、目的に合った評価問題を用意し、正答率だけでなく、速度、費用、安全性、実際の利用者による評価を合わせて確認する。

## まとめ

生成AIの性能向上は、次の掛け算として考えると分かりやすい。

```text
出力品質
  ≒ データの質と量
   × モデルの表現力と構造
   × 学習方法
   × 回答時の推論・検索・ツール
   × それらを実行できる計算資源
```

CPU、GPU、RAM、VRAMの性能向上は重要だが、それだけでAIの知識や推論力が増えるわけではない。強力なハードウェアによって、より大きなモデル、より多くの良質なデータ、長い文脈、複数回の検証といった改善を実行できるようになり、その結果として出力が良くなる。

特に重要なのは、次の4点である。

1. パラメータ数だけでなく、学習データの量と品質とのバランスを取る。
2. 事前学習だけでなく、指示学習やフィードバックで望ましい回答方法を学ばせる。
3. 学習済み知識だけに頼らず、回答時に検索、ツール、検証を使う。
4. 正確さ、速度、費用など、目的に合う複数の指標で性能を判断する。
